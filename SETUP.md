# Profile README — Setup Guide

This repo is your GitHub **profile README** (`alongkon2103/alongkon2103`).
Two of the widgets are generated by GitHub Actions and committed automatically:

| Widget | How it renders | File it produces |
| --- | --- | --- |
| 📊 GitHub Stats + Top Languages | `lowlighter/metrics` action | `github-metrics.svg` (on `main`) |
| 🐍 Contribution Snake | `Platane/snk` action | `github-snake*.svg` (on `output` branch) |

Everything else (Typing, Streak, Visitor Counter, Tech badges) renders from
maintained live services and already works — no setup needed.

---

## 1. Create the repo & push

The repository `alongkon2103/alongkon2103` does not exist yet. Create it on
GitHub (name it **exactly** your username so it becomes your profile README),
then from this folder:

```bash
git remote add origin https://github.com/alongkon2103/alongkon2103.git
git push -u origin main
```

## 2. Enable Actions permissions (REQUIRED for Snake + Metrics)

On GitHub:
`Settings` → `Actions` → `General` →

1. **Actions permissions**: “Allow all actions and reusable workflows”.
2. **Workflow permissions**: select **“Read and write permissions”**.
3. Save.

> The workflows also request `permissions: contents: write` at the job level,
> but the repo-level setting above must allow it too.

## 3. Run the workflows once

`Actions` tab → run each workflow with **“Run workflow”** (`workflow_dispatch`):

- **Generate GitHub Metrics** → commits `github-metrics.svg` to `main`.
- **Generate Contribution Snake** → creates the `output` branch with the snake SVGs.

They also run automatically on every push to `main` and on a schedule
(metrics: daily, snake: every 12h).

## 4. Verify

Refresh your profile at <https://github.com/alongkon2103>. All 4 generated/live
widgets should display. Direct links after the first run:

- Metrics: `github-metrics.svg` (in repo root, `main`)
- Snake (dark): `https://raw.githubusercontent.com/alongkon2103/alongkon2103/output/github-snake-dark.svg`

## Optional: better metrics (private stats / higher rate limit)

`metrics.yml` uses the built-in `GITHUB_TOKEN`, which is enough for public
stats + languages. To include private-contribution counts or avoid rate limits:

1. Create a **classic** Personal Access Token with scopes `read:user` + `repo`.
2. Repo `Settings` → `Secrets and variables` → `Actions` → add secret
   `METRICS_TOKEN`.
3. In `.github/workflows/metrics.yml` change
   `token: ${{ secrets.GITHUB_TOKEN }}` → `token: ${{ secrets.METRICS_TOKEN }}`.
